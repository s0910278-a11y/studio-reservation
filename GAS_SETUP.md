# Google Apps Script (GAS) 連携セットアップ（最後の1ステップ！）

本システムでは、GCPのサービスアカウント等を用いた認証を回避し、かつスプレッドシートへの安全な書き込みを実現するため、あなたのGoogleアカウントを使用した「GAS Web API」を利用しています。

> [!WARNING]
> \*\*【重要】スクリプトの初回実行権限の承認\*\*
>
> 既に `clasp` によるAPIのデプロイや `.env` の設定は自動で完了していますが、Googleのセキュリティ制限により、**初めてスクリプトがスプレッドシートにアクセスする際、所有者（あなた）による明示的な許可が必要です。**

以下の手順を行わないとアクセス拒否エラーとなり、画面やシートにデータが反映されません。**所要時間は1分ほどです。**

## 初回アクセス承認の手順
1. **[Google Apps Scriptダッシュボード](https://script.google.com/home)** をブラウザで開きます。
2. 作成されたプロジェクト『**StudioReservationAPI**』をクリックします。
3. 画面中央上部の「関数を選択」プルダウンメニューから **`doGet`（または `getSheet`）を選択** し、その隣の「**▶ 実行**」ボタンをクリックします。
4. **「承認が必要です」** というポップアップ画面が表示されるので、以下の順でクリックします。
   * `権限を確認`
   * ご自身のアGoogleアカウントを選択
   * `詳細`（左下の小さな文字）
   * `StudioReservationAPI（安全ではないページ）に移動`（一番下のリンク）
   * `許可`（右下）
5. ※実行ログに `Exception`（エラー）と表示された場合でも、**権限さえ許可できれば完了**ですので気にしなくて大丈夫です。

これを行うだけで権限が有効化され、次回のアプリ操作時（またはブラウザをリロードした瞬間）に『予約一覧』と『顧客リスト』のシートが自動でスプレッドシート内に作成され、予約データがリアルタイムで反映されるようになります！

（※合わせて、Next.jsの画面がおかしな場合は `restart_localhost.bat` か `start_localhost.bat` を使って黒いプロンプト画面を再起動してください。）
